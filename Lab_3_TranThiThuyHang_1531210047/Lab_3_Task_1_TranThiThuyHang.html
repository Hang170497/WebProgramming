General

Motivation

While the scope of used hardware in robotics tends to grow, writing reusable robotics software is animportant challenge. ROBOTINOCONTROL is a model-based robotics system trying to tackle this problem by employing MONTIARC[HRR12] for a modular and component-based approach and its extension MONTIARCAUTOMATON[RRW13a, RRW13b]. Note that both build upon the MONTICORE[Kra10]-Framework

Prerequisites

In order to go through the following procedure make sure that you have installed the software packages listed below. Since ROS should run on a Linux machine, we only present the commands to install these packages on Linux. For example: $sudo apt-get install python-pip

How to run

This chapter explains how to run the architecture of ROBOTINOCONTROL and the corresponding user interface. Check out the whole project1 or just ROBOTINOCONTROL2 and the user interface3 and update the repository with Maven. If you need credentials for the repository please contact the staff of the Software Engineering department. Then you can register the project in the ROS build system. You can do that by switching to the root directory of ROBOTINOCONTROL via roscd robotinocontrol and executing rosmake. It is sufficient to do this one time. After that step the project can be built with Maven whenever it is neccessary.

How to use

Having followed the steps of section 1.3 you can now use ROBOTINOCONTROL. Therefore access the user interface in your browser via “http://:8000/web” and log in. You chan choose between four views: Status, Aufgaben, Map, Interaktion.

For further information about technical details of the user interface please have a look at chapter 5
